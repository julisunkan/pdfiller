Build a full-stack web application called **"PDF Form Filler"** using Python Flask and SQLite.

## Core Features

1. Users can upload a PDF file.
2. Extract form fields from the PDF.
3. Display the PDF in the browser using PDF.js.
4. Allow users to:

   * Fill text fields
   * Drag and position fields anywhere on the PDF (like DocuSign)
   * Draw a signature on a canvas and place it on the PDF
5. Generate a filled PDF on the server and allow download.
6. Save reusable templates (field positions + structure) in SQLite.
7. Load saved templates and reuse them.

---

## Technical Requirements

### Backend (Flask)

* Use Flask for server
* Use SQLite for database
* Use pypdf for reading and filling PDFs
* Store uploaded PDFs in `/uploads`
* Store generated PDFs in `/generated`

### Database Schema

Create tables:

* pdfs(id, filename)
* templates(id, pdf_id, field_data JSON)
* (optional) filled_documents(id, template_id, values JSON)

---

## API Endpoints

* POST /upload → upload PDF and return extracted fields
* POST /fill/<pdf_id> → generate filled PDF
* POST /save-template/<pdf_id> → save field layout
* GET /template/<id> → load saved template
* GET /download/<filename> → download generated PDF

---

## Frontend

* Use HTML, CSS, and vanilla JavaScript (no frameworks)
* Use PDF.js to render PDF in a canvas
* Overlay draggable input fields on top of the PDF
* Allow adding:

  * Text fields
